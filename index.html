About Me

빠르게 배우고 성장합니다.

운영 중인 서비스의 프론트엔드 개발 환경을 React와 Next.js로 마이그레이션 하는 과정에 참여하여 기술을 빠르게 습득하고 적용한 경험이 있습니다. 기존 HTML, CSS, Javascript로 구성된 코드를 컴포넌트 기반으로 모듈화하는 과정을 통해 코드 재사용성과 유지보수성을 높일 수 있었습니다. 이 과정에서 프론트엔드 기술의 변천사를 몸소 체험하며 성장할 기회를 가질 수 있었습니다.

특히, 공부한 내용을 바로 서비스에 적용하는 경험을 통해 저의 배움이 개인의 성장에서 그치는 것이 아니라 팀과 서비스에 기여할 수 있음에 큰 즐거움을 느꼈습니다. 그 즐거움이 다시 배움의 동력이 되는 선순환을 통해 꾸준히 발전하는 개발자로 성장할 수 있었습니다.

인터랙션을 통해 사용성을 개선하고,
서비스를 브랜딩합니다.

프론트엔드 개발의 즐거움은 코드로 화면을 구현하는 데 있다고 생각합니다. 특히 작은 움직임으로 사용자 행동에 피드백을 줄 수 있는 마이크로 인터랙션에 많은 관심을 가지고 있습니다. 실제로 웹, 앱을 사용하며 흥미로운 인터랙션을 기억해 두고 서비스에 적용하여 사용성을 개선하였습니다.

또한, 다양한 인터랙션을 통해 서비스를 표현하는 작업에 큰 흥미를 가지고 있습니다. 프리미엄 컨설팅 서비스 런칭 시 스크롤 기반의 인터랙션을 적용하여 서비스 브랜딩에 기여하였으며, 다양한 프로모션 페이지에서 인터랙션을 제안하고 적용하여 사용자의 흥미를 유발한 경험이 있습니다.

같은 목표를 바라보고 성취합니다.

개발의 목적은 서비스가 의도대로 사용자에게 전달되는 것에 있다고 생각합니다. 의도를 담는 과정에서 어긋남이 발생하지 않도록 지속적인 의사소통을 통해 서비스를 바라보는 시선을 맞추며 작업합니다.

이런 과정에서 비개발 직군과의 원활한 의사소통을 위해 스토리북을 도입하여 리액트 컴포넌트 구현 사항을 공유하였고, 또한 협업에 있어 다른 직군에 대한 지식의 필요성을 크게 체감하여 Express로 API 서버를 직접 구축해 보며 백엔드 지식을 학습하였습니다.

개발자이기 전에 함께 목표를 이루어나가는 동료로서 원활한 협업을 위해 끊임없이 소통하며, 필요한 지식이 있다면 적극적으로 학습하여 팀과 서비스에 기여하기 위해 노력하고 있습니다.

Overview

  • 비즈니스 방향 전환에 따른 전면적인 UI 개편 진행.
  • 하드코딩 되어 있던 섹션을 제거하고 API를 통해 받아온 데이터를 통해 자사 포트폴리오 및 상품 리스트 노출.

Results & Lessons Learned

  • 1. UI 개편에 따른 리액트 컴포넌트 개발
    • 버튼, 타이포그라피와 같은 가장 작은 단위의 컴포넌트 개발 시 재사용성을 고려하여 기능과 스타일을 분리. HTML Element의 속성을 모두 Props로 전달하여 ‘스타일이 입혀진 HTML Element’ 처럼 사용할 수 있도록 지원.
    • 아코디언, 탭 메뉴와 같은 컴포넌트 구현 시 컴파운트 패턴을 통해 prop drilling을 피하고 유연하게 UI를 조합할 수 있도록 지원하여 코드 확장성을 높임.
    • 상품 카드 컴포넌트 개발 시 페이지, 섹션에 따라 구성 요소가 조금씩 달라져 컴포넌트의 복잡성 증가. 아토믹 디자인 패턴 도입하여 분리한 컴포넌트를 조합하는 방식으로 코드 복잡성을 낮추고 재사용성을 높임.
  • 2. 브라우저 렌더링 최적화 및 사용자 경험 개선
    • 데이터 로딩 시 스켈레톤 UI를 통해 Layout Shift를 방지하여 브라우저 렌더링 성능을 최적화.
    • 로딩 UI를 통해 현재 페이지에서 진행 중인 작업을 사용자에게 명확히 전달하여 사용자 경험을 개선하고 페이지 이탈 방지.
  • 3. 커뮤니케이션
    • 함께 프로젝트를 진행한 디자이너님께서 HTML, CSS에 대한 이해도가 높아 원활한 커뮤니케이션을 통해 디자인과 개발이 일관되게 적용될 수 있도록 조율할 수 있었음.
    • 타 직군과의 원활한 의사소통을 통해 스토리북을 도입하여 컴포넌트 구현 사항 공유.
    • 해당 프로젝트를 통해 타 직군과의 원활한 의사소통을 위한 지식의 중요성 인식. 이후 개인 프로젝트에서 Express로 API 서버를 구축해보며 백엔드 지식을 학습하여 전반적인 개발 프로세스에 대한 이해도를 높임.

토너먼트 이벤트 페이지 제작

Overview

  • 자사 인테리어 포트폴리오 노출 및 공유하기를 통한 바이럴 마케팅을 목적으로 한 이상형 월드컵 토너먼트 페이지 제작.
  • API를 통해 전달받은 16개의 인테리어 포트폴리오 사진을 랜덤으로 토너먼트 매치한 후 최종 결과 페이지로 랜딩시키는 작업 진행.
  • 결과 페이지에서 최종으로 선택한 인테리어에 사용된 상품 정보와 연관있는 포트폴리오 이미지를 노출하고, 링크, 카카오톡을 통한 공유하기 기능을 지원함.

Results & Lessons Learned

  • 1. 이상형 월드컵 토너먼트 로직 구현
  • 2. 액션 기반의 상태 관리를 통한 렌더링 최적화
    • 초기 작업 시 서로 의존성이 있는 상태를 useState로 관리하여 useEffect가 남발되어 빈번한 리렌더링이 발생.
    • useReducer를 사용하여 토너먼트 중인 포트폴리오 선택, 라운드 변경 등 액션에 따라 상태를 관리.
    • 불필요한 상태를 제거하고 코드 구조를 단순화하며 코드의 가독성을 향상시키고 렌더링 최적화.
  • 3. 사용자 경험 개선
    • 토너먼트 매치 시 사용자가 선택한 포트폴리오를 명확히 인지할 수 있도록 애니메이션 효과를 제안하고, Framer Motion을 사용하여 애니메이션 구현.
  • 4. 목적에 따른 코드 분리를 통한 개발 생산성 향상
    • 이전 프로젝트에서 공유하기 기능을 커스텀 훅으로 분리해두어 해당 프로젝트에서 빠르게 공유하기 기능을 적용하여 작업 시간 단축.
